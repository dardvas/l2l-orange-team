# Структура проекта

```
.
├── app                     настройки приложения
│   ├── dependencies.php    настройки DI-контейнера
│   ├── middleware.php      настройки мидлвар
│   ├── repositories.php    настройки репозиториев данных
│   ├── routes.php          настройки роутинга
│   └── settings.php        настройки приложения
├── composer.json           настройки composer
├── docker-compose.yml      настройки docker-compose
├── logs                    директория с логами
│   └── app.log             основной лог приложения
├── phpunit.xml             настройки PHPUnit
├── public                  публичная директория проекта
│   └── index.php           основная точка входа
├── README.md               этот файл
├── src                     исходный код PHP
│   ├── Application         прослойка между приложением и фреймворком
│   ├── Domain              бизнес-логика приложения
│   └── Infrastructure      инфраструктурный код
├── tests                   тесты PHP, отражают структуру директории src
│   ├── Application         тесты слоя Приложение-Фреймворк
│   ├── bootstrap.php       точка входа тестов
│   ├── Domain              тесты бизнес-логики
│   ├── Infrastructure      тесты инфраструктурного кода
│   └── TestCase.php        базовый класс тестов
├── var                     прочее
│   └── cache               кеш приложения
└── vendor                  сторонние зависимости, устанавливаемые через composer
```

# Запуск локально

Локально запустить проект можно с помощью docker-compose. Для этого нужно установить Docker по инструкции с  [официального сайта](https://docs.docker.com/compose/install/), перейти в директорию с проектом и запустить контейнер: 
```
docker-compose up
```
После этого проект будет доступен по адресу http://localhost:8000
